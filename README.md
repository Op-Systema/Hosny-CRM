# Hosny CRM

تطبيق CRM عربي لإدارة فرص بيع الفرنشايز للمستثمرين. التطبيق مبني فقط بـ HTML وCSS وVanilla JavaScript مع Supabase Auth وDatabase وRLS، ويعمل كملفات static بدون React أو Backend مخصص.

## التشغيل

1. افتح مشروع Supabase.
2. من SQL Editor شغل الملف `sql/schema.sql` كاملًا.
3. تأكد أن `js/supabase.js` يحتوي بيانات مشروعك:
   - `SUPABASE_URL`
   - `SUPABASE_ANON_KEY`
4. افتح `index.html` أو ارفع المجلد على استضافة static.
5. أنشئ أول حساب من صفحة Sign up. أول حساب يصبح `admin` تلقائيًا، وأي حساب بعده يصبح `sales`.

## الأدوار

- `admin`: يرى كل المستثمرين، المستخدمين، الفروع، التقارير، الاجتماعات والمتابعات.
- `sales`: يرى المستثمرين المعينين له أو الذين أنشأهم، ويمكنه إضافة متابعات واجتماعات وتحديث مرحلة البيع.

لتغيير مستخدم إلى مدير يدويًا:

```sql
update public.profiles
set role = 'admin'
where email = 'user@example.com';
```

## النشر

يمكن نشر المشروع على Netlify أو Vercel أو أي static hosting. ارفع محتويات هذا المجلد كما هي، ثم أضف رابط الموقع في إعدادات Supabase Auth:

- Site URL
- Redirect URLs

## Gemini API

لا تضع مفتاح Gemini السري داخل ملفات JavaScript. الملفات static مكشوفة للمتصفح. إذا أردت استخدام Gemini فعليًا، خزّن المفتاح كـ Secret داخل Supabase واستخدم Supabase Edge Function لاستدعاء Gemini من السيرفر، ثم اجعل الصفحة تستدعي الـEdge Function فقط.

المساعد الحالي يعمل بدون AI مدفوع، ويقرأ بيانات Supabase ويولد ردود عربية بقواعد محلية.

## Demo Workflow

1. سجل أول مستخدم ليصبح مديرًا.
2. أضف مستخدم مبيعات من صفحة التسجيل، ثم فعّله أو غيّر دوره من صفحة المستخدمين.
3. أضف مستثمرين وحدد مسؤول المبيعات.
4. حرّك المستثمر بين مراحل خط البيع.
5. أضف متابعة أو اجتماع من ملف المستثمر.
6. افتح التقارير أو المساعد الذكي للحصول على ملخصات من البيانات الحقيقية.
